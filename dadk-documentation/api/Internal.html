ConnectionParameter#

class dadk.internal.ConnectionParameter.ConnectionParameter(annealer_protocol: str = 'http', annealer_address: str = '', annealer_port: int = 8080, annealer_path: str = '', storage_connection_string: str = '', container_name: str = '', qubo_blob_name: str = 'qubo', penalty_qubo_blob_name: str = 'penalty_qubo', inequalities_blob_name: str = 'inequalities', api_key: str = None, proxy: str = None, proxy_port: int = 81, proxy_user: str = None, proxy_password: str = None, timeout: int = 1800, ssl_verify: bool = True, ssl_disable_warnings: bool = False)#

Internal class to store all connection related parameters.

Parameters:

  • annealer_protocol (str) – Protocol for Digital Annealer REST access (‘http’ or ‘https’). GUI via SetAnnealerProfile: Interface / annealer_protocol

  • annealer_address (str) – IP address of the host machine of Digital Annealer. GUI via SetAnnealerProfile: Interface / annealer_address

  • annealer_port (int) – Port of Digital Annealer service on the host machine. GUI via SetAnnealerProfile: Interface / annealer_port

  • annealer_path (str) – Path prefix used as common root for all REST productions. GUI via SetAnnealerProfile: Interface / annealer_path

  • storage_connection_string (str) – Account connection key for Azure Blob. GUI via SetAnnealerProfile: Storage / storage_connection_string

  • container_name (str) – Default container for Azure Blob. GUI via SetAnnealerProfile: Storage / container_name

  • qubo_blob_name (str) – Name of the qubo json in the Azure Blob, default: ‘qubo’. GUI via SetAnnealerProfile: Storage / qubo_blob_name

  • penalty_qubo_blob_name (str) – Name of the penalty_qubo json in the Azure Blob, default: ‘penalty_qubo’. GUI via SetAnnealerProfile: Storage / penalty_qubo_blob_name

  • inequalities_blob_name (str) – Name of the inequalities json in the Azure Blob, default: ‘inequalities’. GUI via SetAnnealerProfile: Storage / inequalities_blob_name

  • api_key (str) – Account key for web service interface. GUI via SetAnnealerProfile: API keys / api_key

  • proxy (str) – Address of proxy server or ‘None’ (default) if no proxy is used. GUI via SetAnnealerProfile: Local network / proxy

  • proxy_port (int) – Port of proxy server addressed by parameter proxy. GUI via SetAnnealerProfile: Local network / proxy_port

  • proxy_user (str) – User of proxy server addressed by parameter proxy. GUI via SetAnnealerProfile: Local network / proxy_user

  • proxy_password (str) – Password of proxy server addressed by parameter proxy. GUI via SetAnnealerProfile: Local network / proxy_password

  • timeout (int) – Timeout for a single http(s)-request in sec. Maximum value of 18.000sec == 5 hours. GUI via SetAnnealerProfile: Timeout / timeout

  • ssl_verify (bool) – Enable/disable SSL verification. GUI via SetAnnealerProfile: SSL / ssl_verify

  • ssl_disable_warnings (bool) – Enable/disable SSL warnings. GUI via SetAnnealerProfile: SSL / ssl_disable_warnings

DAv2Parameter#

OptimizationMethod(Enum)#

class dadk.internal.DAv2Parameter.OptimizationMethod(value)#

OptimizationMethod is an enumeration to represent the kind of annealing executed on DAv2.

Possible values are:

  • annealing, 0: Run DAv2 in mode “annealing”.

  • parallel_tempering, 1: Run DAv2 in mode “parallel tempering”.

SolutionMode(Enum)#

class dadk.internal.DAv2Parameter.SolutionMode(value)#

SolutionMode is an enumeration to represent which results should be returned to the client.

Possible values are:

  • QUICK, 0: Returns overall best configuration only.

  • COMPLETE, 1: Returns all run’s best configuration.

DAv2Parameter#

class dadk.internal.DAv2Parameter.DAv2Parameter(optimization_method: OptimizationMethod = OptimizationMethod.annealing, number_runs: int = 16, number_iterations: int = 1000000, temperature_sampling: bool = True, sampling_parameter: SamplingParameter = None, temperature_start: float = 1000.0, temperature_end: float = 1.0, temperature_mode: int = 0, temperature_interval: int = 1, offset_increase_rate: float = 5.0, pt_temperature_model: str = 'Exponential', pt_replica_exchange_model: str = 'Neighbours', solution_mode: SolutionMode = SolutionMode.COMPLETE, graphics: bool = False)#

Internal class to store all DAv2 related annealing parameters.

Parameters:

  • optimization_method (OptimizationMethod) – Digital Annealer optimization algorithms, ‘annealing’ or ‘parallel_tempering’. GUI: Optimization method

  • number_runs (int) – Number of stochastically independent runs. GUI: Number of runs

  • number_iterations (int) – Total number of iterations per run. GUI: Number of iterations

  • temperature_sampling (bool) – Temperatures GUI: Temperatures

  • sampling_parameter (SamplingParameter) – All sampling related parameters.

  • temperature_start (float) – Start temperature of the annealing process. GUI: Temperature start

  • temperature_end (float) – End temperature of the annealing process. GUI: Temperature end

  • temperature_mode (int) –

    0, 1, or 2 to define the cooling curve:

    0, ‘EXPONENTIAL’:

    Reduce temperature by factor \((1-temperature\_decay)\) every temperature_interval steps

    1, ‘INVERSE’:

    Reduce temperature by factor \((1-temperature\_decay*temperature)\) every temperature_interval steps

    2, ‘INVERSE_ROOT’:

    Reduce temperature by factor \((1-temperature\_decay*temperature^2)\) every temperature_interval steps.

    GUI: Temperature mode

  • temperature_interval (int) – Number of iterations keeping temperature constant. GUI: Temperature interval

  • offset_increase_rate (float) – Increase of dynamic offset when no bit is selected. Set to 0.0 to switch off dynamic energy feature. GUI: Offset increase rate

  • pt_temperature_model (str) – Temperature model for furnace temperature distribution for parallel tempering process. GUI: Temperature model

  • pt_replica_exchange_model (str) – Select replica exchange model for parallel tempering process. GUI: Replica exchange model

  • solution_mode (SolutionMode) – Defines how many solutions should be created. ‘COMPLETE’ returns all run’s best configuration, ‘QUICK’ returns overall best configuration only. GUI: Solution mode

  • graphics (bool) – Switch on graphics output. GUI: Graphics

DAv3cParameter#

class dadk.internal.DAv3cParameter.DAv3cParameter(time_limit_sec: int, target_energy: int, num_group: int, num_solution: int, num_output_solution: int, gs_num_iteration_factor: int, gs_num_iteration_cl: int, gs_ohs_xw1h_num_iteration_factor: int, gs_ohs_xw1h_num_iteration_cl: int, ohs_xw1h_internal_penalty: int | str, gs_penalty_auto_mode: int, gs_penalty_coef: int, gs_penalty_inc_rate: int, gs_max_penalty_coef: int)#

Internal class to store all DAv3c related annealing parameters

Parameters:

  • time_limit_sec (int) – Maximum running time of DA in seconds. Specifies the upper limit of running time of DA. Time_limit_sec should be selected according to problem hardness and size (number of bits). GUI: General / Time limit [sec]

  • target_energy (int) – Threshold energy for fast exit. This may not work correctly if the specified value is larger than its max value or lower than its min value. GUI: General / Target energy

  • num_group (int) – Number of independent optimization processes. Increasing the number of independent optimization processes leads to better coverage of the search space. Note: Increasing this number requires to also increase time_limit_sec such that the search time for each process is sufficient. GUI: General / Number optimizations

  • num_solution (int) – Number of solutions maintained and updated by each optimization process. GUI: General / Solutions per optimization

  • num_output_solution (int) – Maximal number of the best solutions returned by each optimization. Total number of results is num_solution * num_group. GUI: General / Results per optimization

  • gs_num_iteration_factor (int) – Maximal number of iterations in one epoch of the global search in each optimization is gs_num_iteration_factor * number of bits. GUI: Global search / Iterations factor

  • gs_num_iteration_cl (int) – Maximal number of iterations without improvement in one epoch of the global search in each optimization before terminating and continuing with the next epoch. For problems with very deep local minima having a very low value is helpful. GUI: Global search / No improv. cutoff

  • gs_ohs_xw1h_num_iteration_factor (int) – Maximal number of iterations in one epoch of the global search in each optimization is gs_ohs_xw1h_num_iteration_factor * number of bits. Only used when 1Hot search is defined. GUI: Global search / 1Hot iterations factor

  • gs_ohs_xw1h_num_iteration_cl (int) – Maximal number of iterations without improvement in one epoch of the global search in each optimization before terminating and continuing with the next epoch. For problems with very deep local minima having a very low value is helpful. Only used when 1Hot search is defined. GUI: Global search / 1Hot no improv. cutoff

  • ohs_xw1h_internal_penalty (Union[int,str]) –

    Mode of 1hot penalty constraint generation.

    0:

    internal penalty generation off: 1hot constraint as part of penalty polynomial required

    1:

    internal penalty generation on: 1hot constraint not as part of penalty polynomial required

    If 1way 1hot constraint or a 2way 1hot constraint is specified, ohs_xw1h_internal_penalty = 1 is recommended. GUI: Penalty polynomial / 1Hot penalty mode

  • gs_penalty_auto_mode (int) –

    Parameter to choose whether to automatically incrementally adapt gs_penalty_coef to the optimal value.

    0:

    Use gs_penalty_coef as the fixed factor to weight the penalty polynomial during optimization.

    1:

    Start with gs_penalty_coef as weight factor for penalty polynomial and automatically and incrementally increase this factor during optimization by multiplying gs_penalty_inc_rate / 100 repeatedly until gs_max_penalty_coef is reached or the penalty energy is zero.

    GUI: Penalty polynomial / Penalty auto mode

  • gs_penalty_coef (int) – Factor to weight the penalty polynomial. If gs_penalty_auto_mode is 0, this value does not change. If gs_penalty_auto_mode is 1, this initial weight factor is repeatedly increased by gs_penalty_inc_rate until gs_max_penalty_coef is reached or the penalty energy is zero. GUI: Penalty polynomial / Penalty coefficient

  • gs_penalty_inc_rate (int) – Only used if gs_penalty_auto_mode is 1. In this case, the initial weight factor gs_penalty_coef for the penalty polynomial is repeatedly increased by multiplying gs_penalty_inc_rate / 100 until gs_max_penalty_coef is reached or the penalty energy is zero. GUI: Penalty polynomial / Penalty increase rate

  • gs_max_penalty_coef (int) – Maximal value for the penalty coefficient. If gs_penalty_auto_mode is 0, this is the maximal value for gs_penalty_coef. If gs_penalty_auto_mode is 1, this is the maximal value to which gs_penalty_coef can be increased during the automatic adjustment. If gs_max_penalty_coef is set to 0, then the maximal penalty coefficient is 2^63 - 1. GUI: Penalty polynomial / Max penalty coefficient
