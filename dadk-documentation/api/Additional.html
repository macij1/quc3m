classmethod DaoDecoder.register_function(registered_name: str, registered_function: Callable)#

The method registers a custom decoder-function registered_function under the name registered_name.

The registered_function decodes objects of a specific class. The fully qualified classname of that class should be specified as registered_name according to the encoding of the class, see dadk.JSONTools.DaoEncoder.register_function(). The very only parameter of the function is a dictionary of the same structure that the encoder produced. If there are attributes with other encoded classes as values, those are already transformed into the appropriate objects. The function should return an instance of the class with the decoded content of the given dictionary.

Parameters:

  • registered_name (str) – name of decoder in registry.

  • registered_function (Callable) – Function to be registered

DaoEncoder(JSONEncoder)#

class dadk.JSONTools.DaoEncoder(*, skipkeys=False, ensure_ascii=True, check_circular=True, allow_nan=True, sort_keys=False, indent=None, separators=None, default=None)#

DaoEncoder provides methods to convert DADK-objects into json document.

Constructor for JSONEncoder, with sensible defaults.

If skipkeys is false, then it is a TypeError to attempt encoding of keys that are not str, int, float or None. If skipkeys is True, such items are simply skipped.

If ensure_ascii is true, the output is guaranteed to be str objects with all incoming non-ASCII characters escaped. If ensure_ascii is false, the output can contain non-ASCII characters.

If check_circular is true, then lists, dicts, and custom encoded objects will be checked for circular references during encoding to prevent an infinite recursion (which would cause an RecursionError). Otherwise, no such check takes place.

If allow_nan is true, then NaN, Infinity, and -Infinity will be encoded as such. This behavior is not JSON specification compliant, but is consistent with most JavaScript based encoders and decoders. Otherwise, it will be a ValueError to encode such floats.

If sort_keys is true, then the output of dictionaries will be sorted by key; this is useful for regression tests to ensure that JSON serializations can be compared on a day-to-day basis.

If indent is a non-negative integer, then JSON array elements and object members will be pretty-printed with that indent level. An indent level of 0 will only insert newlines. None is the most compact representation.

If specified, separators should be an (item_separator, key_separator) tuple. The default is (’, ‘, ‘: ‘) if indent is None and (‘,’, ‘: ‘) otherwise. To get the most compact JSON representation, you should specify (‘,’, ‘:’) to eliminate whitespace.

If specified, default is a function that gets called for objects that can’t otherwise be serialized. It should return a JSON encodable version of the object or raise a TypeError.

classmethod DaoEncoder.register_function(registered_name: str, registered_function: Callable)#

The method registers a custom encoder-function registered_function under the name registered_name.

The registered_function encodes objects of a specific class. The fully qualified classname of that class has to be specified as registered_name. The very only parameter of the function is an object instance to be encoded. The function returns a dictionary. The dictionary has to contain an attribute "__class__" with fully qualified class name of the encoded class as value. To encode the content of the object the dictionary can contain any collection of further attributes.

Parameters:

  • registered_name (str) – name of function in registry.

  • registered_function (Callable) – Function to be registered

classmethod TSPUtils.create_QUBOS(coordinates: list[tuple[float, float]], factor_rules: float = 100.0, factor_distance: float = 10.0, one_hot: OneHot = OneHot.two_way) → tuple[BinPol, BinPol, BinPol, BinPol]#

Generate TSP-qubos for given coordinates list. In particular, cities are indexed by coordinates index and distances between cities are computed. The full qubo consists of the distance-objective, i.e. the qubo to minimize the overall travelled distance weighted with factor_distance, the time-constraint qubo, i.e. the qubo to enforce that at each time exactly one city is visited weighted by factor_rules, and the city-constraint qubo, i.e. the qubo to enforce that each city is visited at exactly one time weighted by factor_rules. Returned are the full qubo, the time-constraint, the city-constraint and the distance-objective.

Parameters:

  • coordinates (list[tuple[float,float]]) – List of (x,y) coordinates

  • factor_rules (float) – Factor for constraints

  • factor_distance (float) – Factor for objective

  • one_hot (OneHot) – One-hot declaration which is enabled via the generated dadk.BinPol.VarShapeSet

Returns:

Qubo with full TSP problem, qubo with time-constraint, qubo with city-constraint, qubo with distance-objective

Return type:

tuple[BinPol,BinPol,BinPol,BinPol]
