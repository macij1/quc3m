Generate Polynomials and Description#

BinPolGen#

class dadk.BinPolGen.BinPolGen(source: str | None = None)#

BinPolGen defines a simple syntax for the description of polynomials in binary variables. Polynomials described in that syntax can be generated as LaTeX documents, to check the validity of the input as well readable representation. The main purpose is the generation of a Python script, that creates the given polynomial based on BinPol.BinPol.

Grammar in lark syntax with start element polynomial:#

%import common.WS
%ignore WS

bit_array.9 : (factor  bit_factor* "*"?)? ("*"? bit_prod bit_factor* )+

bit_factor.9: bit_mlt
            | bit_div

bit_mlt.9   : "*"? factor

bit_div.9   : "/" factor

bit_prod.9  : BIT_NAME "[" expression ( "," expression )* "]"

polynomial.1: polynomial "+" poly_term      -> add
            | polynomial "-" poly_term      -> sub
            | poly_term

poly_term.0 : poly_factor
            | poly_term "*" poly_factor     -> mlt
            | poly_term "/" expression      -> div

poly_factor.5: bit_array
            | bit_var
            | sum
            | poly_factor "^" NUMBER        -> exp
            | "(" expression ")"
            | "(" polynomial ")"            -> poly_brc
            | term

sum         : "sum" "(" VAR_NAME "," expression "," expression "," polynomial ")"

bit_var     : SLACK_NAME"(" VAR_IMP "," NUMBER ( "," NUMBER )~0..2 ")"

expression.7: expression "+" term           -> add
            | expression "-" term           -> sub
            | term

term.7      : term "*" factor               -> mlt
            | term "/" factor               -> div
            | term "%" factor               -> mod
            | factor

factor.7    : VAR_NAME
            | var_array
            | NUMBER
            | "(" expression ")"            -> brc

var_array   : VAR_NAME "[" expression ( "," expression )* "]"

BIT_NAME    : /#[a-zA-Z][0-9a-zA-Z_]*/

SLACK_NAME  : /@[a-zA-Z][0-9a-zA-Z_]*/

VAR_NAME    : /[a-zA-Z][0-9a-zA-Z_]*/

NUMBER      : /-?\d+/

VAR_IMP     : /sequential|unary|binary|fibonacci/

Example#

To define this polynomial

\[\sum_{i_1=0}^{N - 1} ( 1 - \sum_{i_2=0}^{N - 1} \textbf{x}_{i_1,i_2} )^{2} + \sum_{i_2=0}^{N - 1} ( 1 - \sum_{i_1=0}^{N - 1} \textbf{x}_{i_1,i_2} )^{2} + \sum_{i_1=0}^{N - 1} \sum_{i_2=0}^{N - 1} c_{i_1,i_2}*\textbf{x}_{i_1,i_2}\]

just use the the following code:

sum(i_1, 0, N-1,
    (1 - sum(i_2, 0, N-1,  #x[i_1,i_2]))^2)
+
sum(i_2, 0, N-1,
    (1 - sum(i_1, 0, N-1,  #x[i_1,i_2]))^2)
+
sum(i_1, 0, N-1,
    sum(i_2, 0, N-1,
        c[i_1, i_2] #x[i_1,i_2]))

A BinPolGen object is an anchor to parse and generate polynomials in different representations. If the optional parameter source is specified, it is stored as text; further the source is parsed and the syntax tree is stored internally. This allows generating LaTeX and Python representation of that polynomial.

param Optional[str] source:

source text representing a polynomial in the defined syntax

Methods#

gen_latex#

BinPolGen.gen_latex(tree: Tree | None = None) → str#

The method returns a LaTeX text representation of the polynomial in the defined syntax. If the optional parameter tree is specified, tree is considered otherwise the internally stored tree is used.

tree (Optional[Tree]) – Optional parameter with tree to be generated as LaTeX string

str

gen_pretty_string#

BinPolGen.gen_pretty_string(tree: Tree | None = None, indent: int | None = 2) → str#

The method returns a prettified text representation of the polynomial in the defined syntax. If the optional parameter tree is specified, that tree is considered otherwise the internally stored tree is used.

str

gen_python#

BinPolGen.gen_python(tree: Tree | None = None, function_name: str | None = 'gen_poly', indent: int | None = 4, indent_level: int | None = 0) → str#

The method returns a Python script representation of the polynomial in the defined syntax. If the optional parameter tree is specified, that tree is considered otherwise the internally stored tree is used. The generated script consists of a Python function definition. The name of the function can be specified by the parameter function_name. The indentation of the script can be controlled by two parameters. indent is the number of blanks to be added per level of indentation. With indent_level the initial depth of indentation can be specified. This can be used, if the function should be copied into another function or class definition.

Parameters:

  • tree (Optional[Tree]) – Optional parameter with tree to be generated as string

  • function_name (Optional[str]) – Optional parameter specifies the name of the generated Python function

  • indent (Optional[int]) – Optional parameter allows to set indent for code generation as number of blanks to be used

  • indent_level (Optional[int]) – Optional parameter allows to set initial indentation level

Returns:

Python script

Return type:

str

parse_poly#

BinPolGen.parse_poly(source: str) → Tree#

The source text given as source is parsed. The input text and the created syntax tree are stored internally for generating output as LaTeX and Python. The syntax tree is returned.

Parameters:

source (str) – source text representing a polynomial in the defined syntax

Returns:

syntax tree representing the parsed polynomial

Return type:

Tree

Properties#

property BinPolGen.source: str#

Source text representing a polynomial. Setting the property initiates a parsing and sets the tree attribute with the resulting syntax tree.

property BinPolGen.tree: Tree#

Syntax tree representing a polynomial. (readonly)
