Polynomials#

Polynomials in binary variables and especially QUBOs are used to specify models for the Digital Annealer. This chapter describes all classes to create and use such polynomials. It start with the base class BinPol. Term represents polynomial terms of float coefficient and a product of variables. Inequality is a special linear polynomial specified in an inequality to be less or equal zero, which can be used to define additional constraints and further restrict the search space.

BinPol - Binary polynomials#

class dadk.BinPol.BinPol(var_shape_set: VarShapeSet = None, qubo_matrix_array: ndarray = None, constant: float = 0)#

Objects of the class dadk.BinPol.BinPol represent binary polynomials and are used to define QUBOs which can run on the Digital Annealer.

A dadk.BinPol.BinPol object can be created and afterwards changed in multiple ways in order to define the intended polynomial from current business input data. Beside others the following methods are helpful to define a binary polynomial.

To define the certain terms of the polynomial, use add_term. The method set_term does a similar operation but clears previously set terms based on the same product of variables. This can have unintended effects especially when fixed bits are used, therefore the usage of add_term is recommended in all cases.

Available operations on polynomials are clone add, multiply, multiply_scalar, power, add_variable_penalty, and add_variable. The arithmetic operation on polynomials may be formulated using the operator +, -, * and ** als well.

To evaluate a polynomial the methods compute, get_weights, as_json, and get_json_generator are available.

With the properties degree and N one can check the degree and the number of bits of the polynomial.

An optimization problem has to be formulated as polynomial in binary variables. dadk.BinPol.BinPol supports to construct and combine binary polynomials of any degree. It offers a method reduce_higher_degree to create a quadratic polynomial with the same minima as the original higher order polynomial. This is important, since the Digital Annealer as well as quantum annealers can only process quadratic unconstrained binary optimizations (QUBO). The reduction of a higher order binary optimization (HOBO) can often be avoided by appropriate definition of variables; though this typically increases the number of variables compared to a HOBO formulation, this is in general better than the automatic reduction of degree, which creates additional helper variables as well. The method make_qubo internally calls reduce_higher_degree.

The constructor of the class dadk.BinPol.BinPol creates a new polynomial object. Initially it represents the zero polynomial which is then defined step by step. Providing the parameters qubo_matrix_array and constant it is possible to define a polynomial by the constructor call completely.

The parameter var_shape_set allows to define a symbolic structure for the argument bit vector underlying the polynomial.

If the parameter one_hot is not None it overrides one-hot definitions possibly given in var_shape_set. It can be used to define one-hot groups on the plain linear ordered bit vector especially when symbolic addressing of bits is not used. The parameter one_hot declares bit ranges starting at bit 0 as one-hot sections. 1-dimensional 1-way-1-hot ranges are declare by a list of integers ‘one_way’:[l1, l2, l3, …ln] giving the length of the respective 1-way-1-hot range. 2-dimensional 2-way-1-hot groups are declared as a set of pairs of integers ‘two_way’:[(w_1, h_1), (w_2, h2), (w_n, h_n)]. w_i and h_i stand for width and height of the rectangles and one of rectangle covers w_i * h_i bits. The bit sets are mapped in the order as declared in the list.

Parameters:

  • var_shape_set (VarShapeSet) – Variable shape set of the BinPol instance stored in var_shape_set attribute. Coefficients of the polynomial can be defined via corresponding names and shape consistent indices.

  • qubo_matrix_array (numpy.ndarray) – Coefficient matrix of the polynomial represented by the BinPol instance. Must be a quadratic matrix with data type integer or float.

  • constant (float) – Constant term of the polynomial represented by the BinPol instance.

Define polynomial terms#

add_term#

BinPol.add_term(value: float, *args: Any) → BinPol#

This method adds to the base polynomial a single term (monomial) given by the value of the coefficient and the indices of the intended binary variables. E.g. a term \(42 x_2 x_3 x_7\) is added by one of the following two statements:

polynomial.add_term(42.0, (2, 3, 7))
polynomial.add_term(42.0, 2, 3, 7)
Parameters:

  • value (float) – Value of coefficient of the term (monomial)

  • args (Any) – Indices of binary variables in the product

Returns:

The current object to allow method chaining

Return type:

BinPol

set_term#

BinPol.set_term(value: float, index: Any = ()) → BinPol#

This method is used to define a single term of a polynomial by giving the value of the coefficient and the indices of the intended binary variables. e.g. a term \(42 x_2 x_3 x_7\) is defined by:

polynomial.set_term(42.0, (2, 3, 7))

Caveat: set_term() replaces the coefficient of the specified product of bit variables and might erase terms previously defined. If constant bits are used even formally different terms can refer to identical terms, e.g. for a constant bit \(x_0 = 1\) the terms \(x_1\) and \(x_0 x_1\) are identical; in that case the polynomial \(x_1 + x_0 x_1 = 2 x_1\) is not defined correctly by BinPol().set_term(1, (1,)).set_term(1, (0, 1)) while BinPol().add_term(1, (1,)).add_term(1, (0, 1)) will deliver the expected result. Therefore, the usage of the method BinPol.add_term() is recommended.

Parameters:

  • value (float) – Value of coefficient of the term

  • index (Any) – Indices of the product of binary variables, the empty tuple stands for the constant term

Returns:

The current object to allow method chaining

Return type:

BinPol

add#

This method adds the polynomial other given as parameter to the base polynomial. If the base polynomial contains a VarShapeSet, extension of the latter is only allowed by merging with the VarShapeSet of the added polynomial. All other access to indices exceeding the length of the base VarShapeSet yield an error. If the base polynomial has no VarShapeSet, extension of the maximal index is possible. Moreover, if the added polynomial contains a VarShapeSet, the base polynomial inherits it. Otherwise, addition takes place without VarShapeSet. The polynomial to be added can optionally be multiply by a scalar.

Parameters:

make_clamped#

BinPol.make_clamped(clamping_bits: ndarray | PartialConfig) → BinPol#

This method derives a polynomial from the calling object, which sets some bit variables as constants 0 or 1. The constant values are declared in the clamping_bits array. clamping_bits is expected as numpy array and its size has to coincide with the number of bit variables of the calling BinPol object. Elements 0 or 1 of this array define constant bits with the respective value while entries of value -1 indicate bits that should stay variable in the clamped polynomial. The search space of a clamped polynomial is a subspace of the original seach space. Since the constant bits are simply dropped in the search space of the clamped polynomial, one has to take care of the mapping of bit indices between the two bit vectors of clamped and original polynomial (see BinPol.inflate_clamped_configuration()).

Parameters:

clamping_bits (Union[numpy.ndarray, PartialConfig]) – numpy array defining constant and variable bits or a PartialConfig

Returns:

copy of the current object

Return type:

BinPol

Polynomial evaluation#

compute#

BinPol.compute(bit_values: list[bool] | Dict[str | int, bool], *index: int, **start_stop: Dict[str, int]) → int | float#

This method evaluates the base polynomial at the given list of bit values. Alternatively, one can choose indices in form of integers and/or names from the polynomials variable set. In this case, compute() evaluates only those terms of the base polynomial in which all bits have indices which are contained in the given number of integers or which correspond to a variable array of one of the given names. If no integers or names are given, one can alternatively set the keyword variables start and stop; in this case, compute() evaluates only those terms of the base polynomial in which all bits have indices which are contained in range(start,stop).

Parameters:

  • bit_values (Union[list[bool], Dict[Union[str, int], bool]]) – list or dictionary with index keys containing bit values in {True,False} or {0,1}

  • index (int) – one or more integers and/or strings; only integers in [0, 1, …, number of bits -1] are considered for evaluation; all strings should be names from the polynomials variable set

  • start_stop (Dict[str,int]) – only the keyword arguments start and stop are considered; both arguments should be set to a number in [0, 1, …, number of bits -1]; only integers in range(start,stop) are considered for evaluation

Returns:

value of the base polynomial, evaluated at the given list of bit values; return type equals the type of the base polynomial’s coefficients

Return type:

Union[int, float]

reduce_higher_degree#

classmethod BinPol.reduce_higher_degree(poly: BinPol, penalty_factor: float = 1.0) → BinPol#

This method reduces terms in a BinPol poly of degree higher than 2 by 1 degree. In particular, the method returns a BinPol with the same minima as poly but with reduced degree. For each term in poly which has the highest degree, a product of two bits is replaced by a new variable and the value of the new variable is coupled to the value of the product by a penalty polynomial which attains the value zero iff the new variable has the same value as the product of the two bits. For example, a product \(x_0 \cdot x_1 \cdot x_2\) would be replaced by \(x_0 \cdot y\) where \(y\) is a new bit variable and the additional penalty term \(x_1 \cdot x_2 - 2 \cdot x_1 \cdot y -2 \cdot x_2 \cdot y + 3 \cdot y\) would be added to the returned polynomial to enforce that \(y = 1\) iff \(x_1 \cdot x_2 = 1\).

Parameters:

  • poly (BinPol) – Polynomial to be reduced by one degree if its degree is larger than 2

  • penalty_factor (float) – The penalty polynomials which are added to the new polynomial are multiplied by this factor before being added. Each penalty term penalizes mismatches between replaced products and new bits

Returns:

polynomial with degree one less than or equal to poly

Return type:

BinPol

VarShapeSet#

class dadk.BinPol.VarShapeSet(*var_defs: BitArrayShape | Variable | Category, one_hot_groups: list[OneHotGroup] | None = None)#

dadk.BinPol.VarShapeSet defines an indexing structure for the bits of a BinPol polynomial. Plain BinPol polynomials are defined on a set of bits indexed by a range(N) for some integer N. The VarShapeSet lays a sequence of disjoint named sections over this linear structure. Bits within a section can be addressed by the defined name. With a BitArrayShape a section can be defined as multidimensional array and single bits in the section can be addressed by an appropriate tuple of indices. With a Variable definition the section represents a number encoded in certain bit schemes; in this case it is possible to retrieve the represented value instead of reading single bits.

The constructor creates an addressing scheme based on the BitArrayShape objects given as var_defs. The name attributes of these objects are expected to be unique, and they are used for addressing the respective section of the underlying bit vector of a BinPol polynomial. Duplicate names terminate execution with an ValueError exception.

BitArrayShape objects can define a one_hot property as one_way or two_way. Those objects are expected first in the argument list possibly followed by BitArrayShape objects without one_hot property (i.e. no_way). Wrong order of one_hot and not one_hot arguments terminates execution with an ValueError exception.

For one_hot groups of special shape e.g. discontinued sequences of bits or groups across different BitArrayShape structures the one_hot groups can be specified by a sequence of OneHotGroup objects.

Parameters:

  • var_defs (Union[BitArrayShape, Variable, Category]) – one or more section definitions of type BitArrayShape, Variable or Category

BitArrayShape#

class dadk.BinPol.BitArrayShape(name: str, shape: tuple[int, ...] = None, constant_bits: ndarray = None, one_hot: OneHot = OneHot.no_way, call_back: Callable | None = None, axis_names: list[str] | None = None, index_offsets: list[int] | None = None)#

An object of the class BitArrayShape represents an array structure as part of a bit vector. It allows multidimensional indexed access to the bit variables of a BinPol polynomial. BitArrayShape objects are used inside VarShapeSet objects, which organize index data of a complete bit vector for a polynomial. Bit variables of such polynomials can then be accessed by name and indices according to the shape specified in the BitArrayShape object.

The constructor creates a new variable array object with name and shape as specified. name is an identifier to refer to the variable e.g. when constructing polynomials or retrieving bit vectors. shape specifies the length of all the dimensions of the array.

The optional parameter constant_bits is a numpy array of type int8 with the same shape as the BitArrayShape object itself; it contains values 0, 1 and -1 to indicate bit variables at the respective position of the BitArrayShape object as fixed constant 0, constant 1 or variable with two possible values 0 or 1. Constant bits can be used as normal bits of the BitArrayShape object but when a polynomial is sent to a solver these bits are not part of the constructed polynomial and its corresponding coefficient matrix.

The optional parameter call_back defines a function that is attached as method dadk.Solution_SolutionList.BitArray.evaluate() when objects of type dadk.Solution_SolutionList.BitArray are generated with the method dadk.Solution_SolutionList.Solution.extract_bit_array(). call_back is an optional parameter to evaluate a dadk.Solution_SolutionList.BitArray object in a specific way. call_back can be defined as a method or a function; if intended as a method, the first argument must be of type dadk.Solution_SolutionList.BitArray. call_back has to be defined with a parameter data which must match the dimensions of shape; parameter data can be a keyword argument or otherwise has to be the first parameter (or the second after the bit array instance in case of a method); other keyword parameters are allowed.

The one_hot parameter can be used to define the variable as 1-hot group or collection of 1-hot groups. The default value OneHot.no_way declares the variable without 1-hot characteristic. OneHot.one_way defines 1-way-1-hot constraints for the variable. For a 1-dimensional BitArrayShape all bits of the variable belong to a single 1-hot group and exactly one bit of this variable is True in a valid solution. For multidimensional arrays the variation in the last dimension defines the 1-hot groups, i.e. all collections of bits identical in all but the last dimension form a 1-way-1-hot group. A quadratic 2-dimensional BitArrayShape can be declared as OneHot.two_way. In that case in every column and row of the array exactly one bit of this variable is True in a valid solution. For higher dimensional arrays the last two dimensions have to be identical, then all collections of bits identical in all but the last two dimensions form a 2-way-1-hot group.

The optional parameter axis_names allows the definition of axis names that are used in visualizations of results.

Parameters:

  • name (str) – name of the array

  • shape (tuple[int, ...]) – shape of the index; specify the length of each dimension

  • constant_bits (numpy.ndarray) – numpy array of type int8 with same shape as the previous parameter containing 0 and 1 for constant bits and -1 variable bits

  • one_hot (OneHot) – define variable as one_hot section

  • call_back (Optional[Callable]) – optional routine to evaluate the result of an optimization

  • axis_names (Optional[list[str]]) – Names for the axis.

  • index_offsets (Optional[list[int]]) – index_offsets of the index, specify the index_offsets of each dimension

VariableBase10(Variable)#

class dadk.BinPol.VariableBase10(name: str, start: float = 0.0, stop: float = 100.0, step: float = 1.0, call_back: Callable = None)#

A VariableBase10 object defines a binary polynomial and can be used to represent a numerical value range in a Base10 encoding. The variable is defined by a value range and the corresponding polynomial representation contains a constant corresponding to the start value of the range. The first 10 bits represent \(10^0 * i * step\). Each following group of 10 bits represent the next power of 10. For example a value which is 299 steps away from the start value would be represented as \(2*10^2 + 9*10^1 + 9*10^0\) with the last bit of the first group of 10 bits representing a 9, the last bits of the second group of 10 bits representing a 90 and the third bit of the third group of 10 bits representing a 200.

This constructor creates a new variable object containing all attributes and methods to generate a slack polynomial with the BinPol class using a Base10 encoding. Any integer \(Y\) in \([l_{start}, l_{stop}[\) can be encoded using \(\left\lfloor \frac{d-1}{10^{\left\lfloor \log_{10}(d-1)\right\rfloor}}\right\rfloor + 10 \cdot\left\lfloor \log_{10}(d-1)\right\rfloor + 1\) slack bits \(y_l\) in the following way

\[Y = l_{start} + \sum_{k=0}^{m-1} \sum_{l=0}^{9} l \cdot 10^{k} y_{k, l} + \sum_{l=0}^{n} l \cdot 10^{m} y_{m, l} + \left(d - \sum_{k=0}^{m-1} 9 \cdot 10^{k}\right) y_{m, n+1}\]

where \(m,n\) satisfy \(\sum_{k=0}^{m-1} 9 \cdot 10^{k} + n \cdot 10^{m} < l_{stop} - l_{start} \leq \sum_{k=0}^{m-1} 9 \cdot 10^{k} + (m + 1) 10^{n}\) and \(d=l_{stop} - l_{start}\) and \(b=d+1-a_{M+2}\). This encoding does need two constraint terms

\[H_{Yconstraint,1} = \sum_{k=0}^{m-1} \left( \sum_{l=0}^{9} y_{k, l} - 1 \right)^2\]

and

\[H_{Yconstraint,2} = \left( \sum_{l=0}^{n+1} y_{m, l} - 1 \right)^2 \, .\]

Here, we only show the representation for integers, i.e., \(l_{start}, l_{stop} \in \mathbb{Z}\) and step size=1.
