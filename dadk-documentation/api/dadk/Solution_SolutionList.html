Optimization Solutions#

BitArray#

class dadk.Solution_SolutionList.BitArray(bit_array_shape: BitArrayShape, data: ndarray = None)#

A BitArray object represents a named and structured variable. The structure is taken from the used bit_array_shape of type dadk.BinPol.BitArrayShape. In addition to these base classes a BitArray object contains data to represent a value for the variable. BitArray objects are typically created as part of the solutions and need not be instantiated by dadk users. It provides possibilities for evaluation to translate the result back to business notions.

Parameters:

  • bit_array_shape (BitArrayShape) – Underlying BitArrayShape object.

  • data (numpy.ndarray) – Data array; should contain values in {0,1}

evaluate#

BitArray.evaluate(data: ndarray = None, **kwargs) → int | float#

The method dadk.Solution_SolutionList.BitArray.evaluate() is used to apply the dadk.Solution_SolutionList.BitArray.call_back() function as a method. If call_back is None and the bit array object is of a dadk.BinPol.SlackType type, then dadk.Solution_SolutionList.BitArray.evaluate() evaluates the corresponding slack variable. Otherwise, call_back is applied for evaluation (see dadk.BinPol.BitArrayShape for documentation of call_back). dadk.Solution_SolutionList.BitArray.evaluate() expects a parameter data as well as optional keyword parameters if needed for evaluation of call_back. If data is specified, evaluation is performed on this parameter. Otherwise, the attribute data of the object is used for evaluation.

Parameters:

  • data (numpy.ndarray) – if specified, evaluation is performed on this parameter; dimensions have to match the objects shape attribute

  • kwargs – keyword parameters which are expected by the function call_back; no keyword arguments are expected for the standard call_back routine of slack variables

Returns:

return of call_back function

Return type:

Union[int, float]

Solution#

class dadk.Solution_SolutionList.Solution(energy: int = None, penalty_energy: int = None, frequency: int = 1, configuration: list[bool] = [], var_shape_set: VarShapeSet = None, runs: list[int] = [])#

Solution objects represent a single solution of an optimization. The bit vector in the search space (configuration), the energy and the frequency of adoptions within the complete optimization result can be retrieved. Solutions are returned as part of a SolutionList.

This constructor creates a new solution object containing energy, configuration and frequency of a solution during the annealing process of a solver on a qubo. Solution object also contains the same variable set as the qubo

Parameters:

  • energy (int) – energy of the binary_polynomial at the configuration state

  • penalty_energy (int) – energy of the penalty_binary_polynomial at the configuration state or None (DAv3 or higher)

  • frequency (int) – frequency of the configuration during the solution process

  • configuration (list[bool]) – list with index keys and values in {0,1}. Lists and dictionaries with boolean values will be converted to this form.

  • var_shape_set (VarShapeSet) – Variable shape set of the Solution instance

  • runs (list[int]) – list of integer values, declaring in which run the solutions was generated during a single minimize-call

SolutionList#

class dadk.Solution_SolutionList.SolutionList(solutions: list[~dadk.Solution_SolutionList.Solution] = [<dadk.Solution_SolutionList.Solution object>], solver_times: ~dadk.internal.SolverTimes.SolverTimes | None = None, graphics_data: ~dadk.internal.GraphicsData.GraphicsData | None = None, progress: ~dadk.Solution_SolutionList.Progress | None = None, qubo: ~dadk.BinPol.BinPol | None = None, penalty_qubo: ~dadk.BinPol.BinPol | None = None, inequalities: list[~dadk.BinPol.Inequality] | None = None, da_parameter: ~typing.Any | None = None, scaling_parameter: ~dadk.internal.ScalingParameter.ScalingParameter | None = None)#

dadk.Solution_SolutionList.SolutionList is a helper class used by the solvers to realize objects which contain a list of dadk.Solution_SolutionList.Solution objects, time, graphical and statistical information during the annealing process

This constructor creates a new solution list object containing a list of solution objects, time, graphical and statistical information of the annealing process

Parameters:

  • solutions (list[Solution]) – list of solution objects during the annealing process

  • solver_times (Optional[SolverTimes]) – all measured duration during the annealing process

  • graphics_data (Optional[GraphicsData]) – graphical information of the annealing process such as bit flips

  • progress (Optional[Progress]) – progress information of the annealing process (only DAv3+)

  • qubo (Optional[BinPol]) – the polynomial to be minimized

  • penalty_qubo (Optional[BinPol]) – the optional penalty polynomial to be minimized (only DAv3+)

  • inequalities (Optional[list[Inequality]]) – optional additional inequalities (only DAv3+)

  • da_parameter (Optional[Any]) – annealing parameter used to create this solution

  • scaling_parameter (Optional[ScalingParameter]) – scaling parameter used to create this solution

get_minimum_energy_solution#

SolutionList.get_minimum_energy_solution() → Solution#

From the list of solutions a solution with minimum energy is returned. If there are multiple solutions for the minimum energy then the first one in the list is returned. If penalty_energy is used in the list, then this is the first order sort criterion. Solutions with same penalty_energy are further discriminated by energy as second order sort criterion.

Returns:

solution object in the solution list which has the minimal energy

Return type:

Solution

get_sorted_solution_list#

SolutionList.get_sorted_solution_list(number_of_samples: int = None) → list[Solution]#

The method dadk.Solution_SolutionList.SolutionList.get_sorted_solution_list() returns a copy of the attribute solution list in which the solution objects are sorted by energy. If penalty_energy is used in the list, then this is the first order sort criterion. Solutions with same penalty_energy are further discriminated by energy as second order sort criterion. If number_of_samples is set, the list is truncated to the first number_of_samples solution objects.

Parameters:

number_of_samples (int) – number of solution objects in the list to be returned

Returns:
