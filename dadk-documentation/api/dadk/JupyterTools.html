class dadk.JupyterTools.NumberPlot(data: list[Dict], x_column: str, y_columns: list[str], selection_sets: Dict[str, List] | None = None, title: str = '', add_figure_button: bool = True, left_axis: Dict[str, Any] | None = None, right_axis: Dict[str, Any] | None = None, silent: bool = False)#

The class prepares and displays data as line plot.

The data is expected as a list of dictionaries. Each list item refers to a row of the table. The column headers are given as a list of strings. It is expected that the row dictionaries have keys corresponding to the header fields as specified in y_colums. The optional parameter selection_sets is a dictionary of lists. The key refers to the group name and will be displayed as a button while the values are sub lists of y_columns. The corresponding line plots can be displayed by clicking the button. The optional parameters left_axis and right_axis describe the usage of the two y-axes left and right. The label and label_offset attributes determine the content and layout of axis description. The filter attribute is a function applied to the values of a line as list. It decides whether the line is painted with respect to the axis. It is possible to ignore line completely or to paint them related to either one or to both axis. In the default setting all lines with absolute value exceeding 10 are painted and scaled with the left axis and all others with the right axis. This behaviour can be achieved by the following settings:

self.left_axis = {'label': 'time [s] (>=10)', 'label_offset': '50px', 'filter': lambda x: abs(max(x)) >= 10}
self.right_axis = {'label': 'time [s] (<10)', 'label_offset': '50px', 'filter': lambda x: abs(max(x)) < 10}

NumberTable#

class dadk.JupyterTools.NumberTable(data: list[Dict | object], headers: list[str], css_styles: list[Dict[str, str]] = None, type_converters: list[Callable] | None = None, summary_definition: tuple[Dict[str, Any]] | None = None, download: bool = True, file_name: str = 'my_table', sort_control: bool = True, sort_column: str = None, sort_direction: int = 0, silent: bool = False)#

The class prepares and displays a table. For numerical columns an average and a sum is calculated by default and displayed.

The data is expected as list of dictionaries or objects. Each list item refers to a row of the table. The column headers are given as a list of strings. It is expected that the row items have corresponding entries. In the case of dictionaries the header fields are identical to keys in the dictionary. In the case of objects, the header strings are transformed to attribute name (i.e. lower case and blanks replaced by ‘_’). With the optional parameter css_styles a structure parallel to data with cell-wise style definition can be specified; e.g. an entry {'col_header': 'background-color: blue;'} will display a blue background in the column col_header of the respective line.

The optional argument type_converters contains type converting functions for the columns like the str, int or float function. Missing items are assumed as float.

summary_definition is a list of dictionaries. Each dictionary contains the function that will be applied to the list of values of a column. The result will be displayed in the table with label given as label. If a color is specified, the value generated by function will be searched in the respective table column and highlighted with that color. In this dictionary the key function must be specified with the function as value that will operate on the list of column values in the table. The comparison of the cell value and the aggregate value of the column for coloring can be customize by a function specified with the optional key compare. This function gets the cell value and aggregate value as argumnts and returns True iff the cell has to be colored.

The following example for summary_definition defines a column wise minimum, maximum, average, sum and median function.

summary_definition = (

    {'name1': 'minimum', 'color': 'green', 'label': 'Min', 'arg_types': [int, float], 'function': min},
    {'name2': 'maximum', 'color': 'red', 'label': 'Max','arg_types': [int, float], 'function': max},
    {'name3': 'average', 'color': 'black', 'label': 'Avg', 'arg_types': [int, float],
        'function': lambda x: sum(x) / len(x) if len(x) > 0 else 0},
    {'name4': 'sum', 'color': 'black', 'label': 'Sum', 'arg_types': [int, float],
        'function': sum, 'compare': lambda x, colresult: False},
    {'name5': 'median', 'color': 'yellow', 'label': 'Median',  'columns': ['Integer 2'],
        'function': median, 'compare': lambda x, col_result: x <= col_result}
)
Parameters:

  • data (list[Union[Dict,object]]) – List of table rows given as dictionaries or objects

  • headers (list[str]) – List of column header strings

  • css_styles (list[Dict[str,str]]) – List corresponding to data of dictionaries containing CSS definitions for columns

  • type_converters (Optional[list[Callable]]) – Optional list of type converting functions for columns

  • summary_definition (Optional[tuple[Dict[str,Any]]]) – List of dictionaries with function to be operated on each column and result highlighting

  • download (bool) – Whether to display button for content download

  • file_name (str) – Name of download file

  • sort_control (bool) – Add sort control to control panel of table

  • sort_column (str) – Column with sort key, None means no sort

  • sort_direction (int) – Do sorting in ASCENDING, ASCENDINGABS, DESCENDING or DESCENDINGABS order

  • silent (bool) – Whether to display content directly upon construction

TaggedGuiControl#

class dadk.JupyterTools.TaggedGuiControl(*widget: Widget, get_persistent_tags: Callable[[], Dict[str, List]] = None, set_persistent_tags: Callable[[Dict[str, List]], None] = None, before_update: Callable = None, after_update: Callable = None, two_line_creator: bool = True, silent: bool = True)#

This class supports tagged setting of values for a set of other jupyter widgets named the “controlled widgets”. The object itself is controlled by a creator and an applier control.

The creator consists of a text input field for a tag name and create and delete button. Pressing the create button (plus icon) creates a new tag representing the current values of the controlled widgets. If the tag already exists, it is updated with the current values of the controlled widgets. Pressing the delete button (trash icon) removes the tag from the list of available tags.

The applier contains a button for each available tag. Pressing the button applies the stored values to the controlled widgets. The applier displays the currently available tags and is updated once a new tag is created or an existing one is deleted from the creator control.

The tagged GUI control can control one or multiple widgets; these controlled widgets are specified as first parameters. get_persistent_data is an optionally defined function or method, that returns a dictionary with tag values. Each tag is defined as a text key in the dictionary. The value is a list consisting of values of the controlled widgets in the same order as they are have been specified in this constructor call. If specified then get_persistent_tags is called once to initialize the object with stored tags. set_persistent_tags is the opposite direction. It is called with a dictionary of tagged values for the controlled widgets and it is expected to store this dictionary so that is can be retrieved later by a call of get_persistent_tags. The function is called everytime the set of tags is modified.

Parameters:

  • widget (widgets.Widget) – One or multiple widgets to be controlled

  • get_persistent_tags (Callable[[],Dict[str,List]]) – Optional function or method to retrieve persistent data as dictionary

  • set_persistent_tags (Callable[[Dict[str,List]],None]) – ptional function or method to store persistent data from given dictionary

  • before_update (Callable) – Function or method to be called before tagged values are applied

  • after_update (Callable) – Function or method to be called after tagged values are applied

  • two_line_creator (bool) – Use 2 lines for creator text field and action buttons

  • silent (bool) – If set to False, creator and applier are displayed in output context of object creation

WidgetDefinition(dict)#

class dadk.JupyterTools.WidgetDefinition#

A widget is a GUI component that is defined by a dictionary using the following key value pairs.

General keys (applicable to all wigdets)

  • name : mandatory, string forming a valid Python identifier.

  • label : optional, label of the field, default is name, can include LateX code like ‘$A_1$’.

  • description : optional, tooltip text for the field. If missing, label will be used.

  • visible : optional, default is True, used to hide/show a widget.

  • disabled : optional, default is False, used to disable a widget.

  • tags : optional, list of tag-names. Field will be invisible, if in tagged mode none of those tags are set.

  • type : mandatory, defines the widget type, possibilities are listed below.

  • value : optional, default value of the field.

Special keys (Applicable depending on type)

  • min : mandatory, minimum value (included)

  • max : mandatory, maximum value (included)

  • step : optional, increment value, default is 1

  • base : mandatory, defines base for logarithmic scale

  • format : optional, format-string to show selected value on the right side, default is ‘.2f’,

  • rows : mandatory, number of rows, minimum 1

  • columns : mandatory, number of columns, minimum 1

  • headers : optional, a list of strings, to be shown as column headers

  • options : mandatory, list of entries for selection, like [1, 2, 3] or list of 2-tupels, (display text, internal value), like [(‘One’, 1), (‘Two’, 2), (‘Three’, 3)]

Widget types (Determine the representation and behaviour)

  • text : single line text entry

  • text_area : multi-line text entry

  • int_bounded : integer number input field

  • int_slider : integer input with a slider

  • int_range_slider : integer range input by two sliders returns tuple of integers

  • int_array : Rectangular table of integer input fields

  • float_bounded : float number input field

  • float_slider : float input with a slider

  • float_log_slider : float input with a slider on logarithmic scale

  • float_range_slider : float range input by two sliders returns tuple of float

  • float_array : Rectangular table of float input fields

  • select : dropdown list

  • select_multiple : dropdown list, where you can select multiple entries

  • select_slider : selection done with a slider

  • select_range_slider : selection range input by two sliders returns all item between the sliders

  • radio : a group of radio-buttons

  • check : two buttons True and False, and you can only select one (no parameter options)

  • file_selector : select single file, on the system where the jupyter-server is running

  • directory_selector : select single directory, on the system where the jupyter-server is running

  • date_picker : select a date as DateTime object

  • time_picker : select a time as three integer values for hour, minute and second

  • color_picker: to select a color

Definition of widgets in the form of plain dictionaries.

Below you find the available widgets, grouped by functionality and listed by their type, along with a description and a visualization.

Text-input widgets:

  • text : single line text entry

  • text_area : multi-line text entry

  • additional parameters

    • None

Integer-input widgets

  • int_bounded : integer number input field with spinner buttons on the right side. You can define min, max and step of the spinner field

  • int_slider : integer input with a slider, between min and max, with increment step

  • int_range_slider : similar to int_slider, but with two values, so you can define a range, value is a 2-tuple of integers

  • additional parameters

    • min : mandatory, minimum integer value (included)

    • max : mandatory, maximum integer value (included)

    • step : optional, increment integer value, default is 1

Integer-array-input widgets
  • in addition to the fields from the single integer-widgets you have the following additional parameters.

    • rows : mandatory, number of rows, minimum 1

    • columns : mandatory, number of columns, minimum 1

    • headers : optional, a list of strings, to be shown as column headers

Float-input widgets

  • float_bounded : similar to int_bounded, but with float number instead of integer number

  • float_slider : similar to int_slider, but with float number instead of integer number

  • float_log_slider : similar to float_slider, but with logarithmic scale, additional parameter base for the logarithmic scale

  • float_range_slider : similar to int_range_slider, but with float number instead of integer number, value is a 2-tuple of floats

  • additional parameters

    • min : mandatory, minimum float value (included)

    • max : mandatory, maximum float value (included)

    • step : optional, increment float value, default is 1.0

    • format : optional, format-string to show selected value on the right side, default is ‘.2f’,

Float-array-input widgets
  • in addition to the parameters from the single float-widgets you have the following additional parameters.

    • rows : mandatory, number of rows, minimum 1

    • columns : mandatory, number of columns, minimum 1

    • headers : optional, an list of strings, to be shown as column headers

Select widgets

  • select : dropdown list, where you can select one entry (also an example added where internal value is different from visual presentation)

  • select_multiple : dropdown list, where you can select multiple entries

  • select_slider : selection done with a slider

  • select_range_slider : similar to slider, but now you can select a range, like t-shirt sizes from “s” to “l”

  • radio : a group of radio-buttons, where you can only select one

  • check : two buttons True and False, and you can only select one (no parameter options)

  • additional parameters

    • options : mandatory, list of entries for the dropdown list, like [‘A’, ‘B’, ‘C’]. Could also be a list of 2-tupels, where the first entry of each tuple is the visual text, and the second entry is the internal value, like [(‘One’, 1), (‘Two’, 2), (‘Three’, 3)]

File-select widgets

  • file_selector : select single file, on the system where the jupyter-server is running

  • directory_selector : select single directory, on the system where the jupyter-server is running

Special widgets

  • date_picker : in order to select a date

  • time_picker : in order to select a time (just 3 integer fields in a row, first field is ‘hour’, second field is ‘minute’ and finally third field is ‘second’)

  • color_picker: to select a color

Structure widgets

  • accordion : Usually all widgets are added to a single horizontal group, and are visible in the order of their declaration. In case of many widgets, you can structure your input fields in multiple groups. Therefore, you can add additional grouping information. A single group starts with {‘type’:’begin_group’, ‘label’: ‘Text’} (optionally you can add tags here), and ends with {‘type’:’end_group’}. Please notice, that this is the only widget element without a name attribute. You can have multiple begin_group elements before and end_group, so the framework will generate an accordion for you. An example of an accordion can be seen in the example below. Groups (or accordions) cannot be nested!

WidgetGui(HBox)#

class dadk.JupyterTools.WidgetGui(**kwargs: Any)#

This class provides a widget based GUI for a Jupyter notebook to drive a function or method call.

The parameters and the respective GUI elements for their specification are defined in the list parameters. Each list item is a dictionary with keys name, label, description, value, type, min, max, step, options or tags. name is the name of a parameter of the launched function. label and description are used in the GUI to mark the input element for the parameter. type is a widget type to be used and has one of the following string values: text, text_area, float_bounded, int_bounded, float_slider, float_log_slider, float_range_slider, int_slider, int_range_slider, select, select_slider, select_range_slider, radio, select_multiple, check, checkbox, directory_selector, file_selector, date_picker, color_picker for the built-in widgets. The type custom refers to an own customized widget. In this case an additional attribute class contains the name of the widget class as it is named as attribute within the widgets object. A custom widget is registered by

@widgets.register
class MyCustomWidget():
...
WidgetGui.register_widget_class(MyCustomWidget, 'MyCustomWidgetReference')

Then MyCustomWidgetReference is used as class attribute in the parameter definition.

The value entry defines the initial value and its type has to be consistent with the parameter of the function and the selected widget type. Related to the specified type the entries for the keys min, max, step, options or attributes that are defined for custom widgets define the widget behaviour.

The types begin_group and end_group allow the structuring of the widget list by accordions. The begin_group starts an accordion and a new group and the end_group ends the last group and the complete accordion. Dictionaries with type begin_group and end_group must not contain a name key.

tags can specify a list of tags to display or suppress the widget as requested by the argument filter_by_tags.

fixed_parameters define as directory function parameters to be used with fixed values in each call.

persistent_setting can be None or a dictionary with keys file and name. In the second case file specifies the path and name of a JSON file, which contains under the object hierarchy ["parameter_definition"]["<name>"] a dictionary for the parameter values. If persistent_setting is defined after pressing the call button the given values are made persistent. The read_only parameter can block this and preserve the persistent settings.

Callback functions to handle the execution time of the function and the result can be defined. The function specified as update_gui is taken as method of the WidgetGui object and is called after initialization and after each call of the function and it can do updates in the GUI.

The GUI offers a call button to launch the given function (function or method). If break_after_statement is not None it is used as a regular expression to identify statements in the function definition. The execution can be interrupted after these statements. This is controlled by the buttons debug, resume and finish. A clear button allows to clear the output field.

Parameters:

  • function (Callable) – Function or method to be launched

  • parameter_definition (list[Dict]) – List of dictionaries

  • fixed_parameters (Dict) – Fixed parameters not changeable by GUI

  • filter_by_tags (list[str]) – List of string tags to display only parameters tagged with one of the members

  • persistent_setting (Dict) – Dict with file and name keys (see above)

  • name (str) – name, needed for parameter logging in batch mode

  • parameter_logging (ParameterLogging) – Define which parameters should be logged in batch mode, see dadk._Constants.ParameterLogging

  • tag (str) – Tagging, in case of multiple dialogs on a single tab

  • read_only (bool) – Read persistent data for initialization but don’t change it

  • persistent_read_function (Callable) – Use the read method from Optimizer, in order to use proper decoder

  • persistent_write_function (Callable) – Use the write method from Optimizer, in order to use proper decoder

  • execution_time_receiver (Callable) – Callback function for time handling called with timedelta for execution time

  • result_receiver (Callable) – Callback function for result handling called with result of launched function

  • update_gui (Callable) – Callback method to update GUI after __init__ and after call is completed

  • break_after_statement (str) – Regular expression to define statements for breakpoints

  • multiple_output (bool) – Will we have a single output, or multiple (multiple is default for build and solve in case of a compose_model dadk.Optimizer.OptimizerModel)

  • before_call_function (Callable) – Enable additional function to be executed before the call button-click is executed, for example for additional input validation

  • wrapper_function (Callable) – Wrapper function around the main call function, needed by the Optimizer framework

  • call_display (bool) – Display GUI immediately after creating object

  • tab_width_percent (float) – Relative width of launch panel

  • optimizer_model (OptimizerModel) – OptimizerModel object if used in the context of Optimizer

  • style (Dict) – Style of this widget.

ParameterLogging(Enum)#

class dadk.JupyterTools.ParameterLogging(value)#

ParameterLogging is an enumeration to represent the verbosity of the Optimizer running in batch-mode.

Possible values are:

  • NONE, 0: Don’t log any parameters from the GUI.

  • VISIBLE, 1: Log only parameters where the GUI-field is visible.

  • ALL, 2: Log all parameters from the GUI.

prolog#

dadk.JupyterTools.prolog(follow_ups: list[~typing.Dict[str, ~typing.Any]] = [{'action': <built-in function print>, 'label': 'Automatic start in {:.0f} seconds', 'name': 'auto_start', 'seconds': 5, 'style': 'info'}, {'action': None, 'label': 'Stop', 'name': 'stop', 'style': 'danger'}, {'action': <built-in function print>, 'label': 'Start immediate', 'name': 'immediate'}])#

The function creates a dialog as intro (prolog) and selection of possible follow-up actions. It uses a progress bar for count down and triggers an automatic action when zero is reached. Additional buttons allow alternative actions or stop of count down. The GUI and the actions are defined in the follow_ups list.

Each follow-up is defined by a dictionary with name as an obligatory attribute. The first entry of the list defines the count-down. The seconds attribute set the duration of the countdown. The label entry is displayed in front the progress bar and may contain a format pattern like {:d} to display the remaining seconds.

The 2nd ff. list items define control buttons. Their labels must be plain test without formatting pattern.

All follow-up have an action, which is a callback function that must accept a string argument. When the follow-up becomes active (count down at zero or button pressed) then the function is called with the name of the respective follow-up. This allows use of common function for follow-up processing. If action is None no further processing is launched.

The style attribute sets the colors and allows the Jupyter flavours ‘success’, ‘info’, ‘warning’, ‘danger’ or ‘’.

Before starting the follow-up processing the prolog GUI disappears.

Parameters:

follow_ups (list[Dict[str,Any]]) – list of follow-up activities (see description)
