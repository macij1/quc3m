Quadratic Unconstrained Optimization#

DADK supports several features to simplify programming of QUBO terms which appear in many optimization problem and to improve performance for the generation of such QUBO terms.

Let var_shape_set be programmed and fixed as in the previous section:

from dadk.BinPol import *

days = ['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday']
D = len(days)

shifts = ['Early', 'Late', 'Night']
S = len(shifts)

workers = ['Adam1', 'Bob1', 'Charly1', 'Dave1', 'Earl1',
           'Adam2', 'Bob2', 'Charly2', 'Dave2', 'Earl2',
           'Adam3', 'Bob3', 'Charly3', 'Dave3', 'Earl3',
           'Adam4', 'Bob4', 'Charly4', 'Dave4', 'Earl4']
W = len(workers)

blocked = [('Monday', 'Night', 'Adam1'),
           ('Wednesday', 'Late', 'Charly2'),
           ('Friday', 'Early', 'Earl2')]

constant_bits_x_bits = np.full((D, S, W), -1, dtype=np.int8)
for day, shift, worker in blocked:
    constant_bits_x_bits[days.index(day), shifts.index(shift), workers.index(worker)] = 0

x_bit_array_shape = BitArrayShape(name='x', constant_bits=constant_bits_x_bits)

demand = {
    'Monday': {'Early': 2, 'Late': 5, 'Night': 3},
    'Tuesday': {'Early': 1, 'Late': 5, 'Night': 3},
    'Wednesday': {'Early': 2, 'Late': 5, 'Night': 3},
    'Thursday': {'Early': 1, 'Late': 5, 'Night': 3},
    'Friday': {'Early': 2, 'Late': 5, 'Night': 1}}

one_hot_groups_list = [OneHotGroup(('x', d, s, None))
                       for d, day in enumerate(days)
                       for s, shift in enumerate(shifts)
                       if demand[day][shift] == 1]

y_bit_array_shape = BitArrayShape(name='y', shape=(W,))

var_shape_set = VarShapeSet(x_bit_array_shape,
                            y_bit_array_shape,
                            one_hot_groups=one_hot_groups_list)
BinPol.freeze_var_shape_set(var_shape_set)

Exactly 1-bit-on constraints#

One such feature is the method add_exactly_1_bit_on which can be used to generate constraints related to one hot groups. Consider our example of the previous chapter where some shifts \(s\in \mathcal{S}\) on a given day \(d\in \mathcal{D}\) require exactly one worker, i.e., \(a_{d,s}=1\). For such shifts we require \(\left(1-\sum_{w\in \mathcal{W}}x_{d,s,w}\right)^2 = 0.\)

With fixed var_shape_set as programmed in the previous section, this type of constraint can be easily programmed as

weight = 2000
qubo = BinPol()
for d, day in enumerate(days):
    for s, shift in enumerate(shifts):
        if demand[day][shift] == 1:
            qubo.add_exactly_1_bit_on(bits=[('x', d, s, w) for w, worker in enumerate(workers)], scalar=weight)

Here, the constraints are multiplied with a weighting factor weight.

Note

The method generate_penalty_polynomial can be used to simplify the code above.

Exactly n-bits-on constraints#

The more general method add_exactly_n_bits_on allows to directly add the constraints also for days and shifts with more than one required worker:

weight = 2000
qubo = BinPol()
for d, day in enumerate(days):
    for s, shift in enumerate(shifts):
        qubo.add_exactly_n_bits_on(n=demand[day][shift],
                                   bits=[('x', d, s, w) for w, worker in enumerate(workers)],
                                   scalar=weight)

Note

For days and shifts for which a one way one hot group over the workers has been defined, the corresponding one hot constraint has been added to qubo by either of the code examples above.

Most 1-bit-on constraints#

Similarly, the constraint \(\sum_{d\in \mathcal{D}}\sum_{w\in \mathcal{W}}\left[\left(\sum_{s\in \mathcal{S}}x_{d,s,w} - 1\right)\sum_{s\in \mathcal{S}}x_{d,s,w}\right]\) which enforces that each worker can work in at most one shift per day is programmed as

for w, worker in enumerate(workers):
    for d, day in enumerate(days):
        qubo.add_most_1_bit_on(bits=[('x', d, s, w) for s, shift in enumerate(shifts)],
                               scalar=weight)

Other constraints#

The constraint to set \(y_w\) to one for \(w\in \mathcal{W}\) if \(x_{d,s,w}=1\) for at least one \(d\in \mathcal{D}\) and \(s\in \mathcal{S}\), i.e., if at least one shift is assigned to the worker is formulated as \(\sum_{w\in \mathcal{W}}\left(1 - y_w\right)\left(\sum_{d\in \mathcal{D}}\sum_{s\in \mathcal{S}}x_{d,s,w}\right)\).

This constraint can be programmed efficiently by multiplying all brackets in the formula and adding all linear and quadratic terms separately to qubo instead of first computing the single linear polynomials as separate BinPol objects, and multiplying them and adding them to qubo:

for w, worker in enumerate(workers):
    for d, day in enumerate(days):
        for s, shift in enumerate(shifts):
            qubo.add_term(weight, ('x', d, s, w))
            qubo.add_term(-weight, ('x', d, s, w), ('y', w))

Note that we have added linear and quadratic terms with the add_term method.

Optimization Objectives#

There are many useful objectives for the example in this section depending on the concrete use case. For example, it might be useful from an economic point of view to use the least possible number of workers in order to save budget, i.e., to minimize \(\sum_{w\in \mathcal{W}}y_{w}.\) This objective is for example programmed as

weight = 1800
for w, worker in enumerate(workers):
    qubo.add_term(weight, ('y', w))

Alternatively, amounting to the worker’s well-being, it might be of interest to distribute the work evenly among the workers by minimizing \(\sum_{w\in \mathcal{W}}\left(\frac{D\cdot S}{W} - \sum_{d\in \mathcal{D}}\sum_{s\in \mathcal{S}}x_{d,s,w}\right)^2\), i.e., minimizing the difference between the number of shifts assigned to a worker and the average number of shift per worker. This objective can be programmed in the following way

nr_shifts_per_worker = sum([demand[day][shift] for d, day in enumerate(days) for s, shift in enumerate(shifts)]) / W
weight = 400
for w, worker in enumerate(workers):
    qubo_tmp = BinPol()
    qubo_tmp.add_term(nr_shifts_per_worker)
    for d, day in enumerate(days):
        for s, shift in enumerate(shifts):
            qubo_tmp.add_term(-1, ('x', d, s, w))
    qubo_tmp.power2()
    qubo.add(qubo_tmp, weight)

Moreover, one should decide for either of the objectives above, since minimizing the number of workers deforms the average number of workers per shift. In particular, if the second objective is of interest, one can drop the \(y_w\) variables for \(w\in \mathcal{W}\) all together along with constraints including them.

Note

As the name Quadratic Unconstrained Binary Optimization suggests, all objectives and constraints are summed up within one formula. In order to obtain a formula with minimizing states that fulfill all constraints, those constraints have to be multiplied with a sufficiently large weight. At the same time, this weight should not be too large, otherwise the objective is neglected by Digital Annealer.

Moreover, Digital Annealer works integer based, so the integer precision of the weights is not infinite. In practice, the weight multiplied to constraint QUBOs often does not need to be larger or equal than a proven necessary weight but can be smaller and determined by a few experiments.

Final QUBO#

The final QUBO in the example with objective to minimize the number of workers is \(\begin{align}1800\cdot\sum_{w\in \mathcal{W}}y_{w} &+ 2000\cdot\sum_{d\in \mathcal{D}}\sum_{s\in \mathcal{S}}\left(a_{d,s}-\sum_{w\in \mathcal{W}}x_{d,s,w}\right)^2 + 2000\cdot \sum_{d\in \mathcal{D}}\sum_{w\in \mathcal{W}}\left[\left(\sum_{s\in \mathcal{S}}x_{d,s,w} - 1\right)\sum_{s\in \mathcal{S}}x_{d,s,w}\right]\\ &+ 2000\cdot \sum_{w\in \mathcal{W}}\left(1 - y_w\right)\left(\sum_{d\in \mathcal{D}}\sum_{s\in \mathcal{S}}x_{d,s,w}\right).\end{align}\)

For the objective to spread work evenly among the workers, the final QUBO is \(\begin{align}400\cdot\sum_{w\in \mathcal{W}}\left(\frac{D\cdot S}{W} - \sum_{d\in \mathcal{D}}\sum_{s\in \mathcal{S}}x_{d,s,w}\right)^2 &+ 2000\cdot\sum_{d\in \mathcal{D}}\sum_{s\in \mathcal{S}}\left(a_{d,s}-\sum_{w\in \mathcal{W}}x_{d,s,w}\right)^2\\ &+ 2000\cdot \sum_{d\in \mathcal{D}}\sum_{w\in \mathcal{W}}\left[\left(\sum_{s\in \mathcal{S}}x_{d,s,w} - 1\right)\sum_{s\in \mathcal{S}}x_{d,s,w}\right]. \end{align}\)
