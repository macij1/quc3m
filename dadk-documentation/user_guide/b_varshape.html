Structured Variables#

Before a combinatorial optimization problem in business language is programmed and optimized, it needs to be formulated mathematically. In particular, a set of binary decision variables typically structured as array with indices is defined to formulate the optimization task.

Variables#

Consider for example a mathematical problem which contains a set of variables \(x_i \in \{0, 1\}\) with \(i=0,\ldots,N\) with \(N>0\). The DADK supports to keep the variables \(x_i\) also on the programming side. This is done within a VarShapeSet object. In particular, each variable set is defined by a BitArrayShape object with a variable name and index dimensions which is then given to a VarShapeSet where all variable definitions are managed. For \(x_i\) with \(i=0,\ldots,N=9\) a BitArrayShape with a one dimensional index and \(N=10\) entries can be implemented as

from dadk.BinPol import *

N = 10
x_bit_array_shape = BitArrayShape(name='x', shape=(N,))

The VarShapeSet for the program is defined as

var_shape_set = VarShapeSet(x_bit_array_shape)
BinPol.freeze_var_shape_set(var_shape_set)

The last statement defines var_shape_set for the subsequent code and every class in the DADK which expects a VarShapeSet object will automatically be provided var_shape_set.

Note

A VarShapeSet object can take more than one BitArrayShape. This allows for a very easy and intuitive implementation of optimization problems which require more than one type of decision variable; for examples see Advanced Usage - Structured Variables.

Binary Polynomials#

The DADK representation of a binary polynomial is done with a BinPol class. With this the basic polynomial \(\sum_{i=0}^{N-1} x_i\) can be programmed as a BinPol object as

qubo = BinPol()
for i in range(N):
    qubo += Term(1, ('x', i))

which can easily be linked to the original mathematical formula.

Note

Both BinPol and Term expect a VarShapeSet object. In the example this is not necessary since we fixed var_shape_set with freeze_var_shape_set.

Alternatively the terms of the polynomial can be added using the method add_term. This code is faster since it abstains from the instantiation of Term objects and should be preferred for larger QUBOs.

qubo = BinPol()
for i in range(N):
    qubo.add_term(1, ('x', i))

The method as_latex returns a string with the latex code for the binary polynomial qubo. For example,

qubo.as_latex()

returns the string x_{0} + x_{1} + x_{2} + x_{3} + x_{4} + x_{5} + x_{6} + x_{7} + x_{8} + x_{9}.

You will learn more about quadratic unconstrained optimization and the BinPol class in the next chapter.

Note

The DADK can not only handle binary variables but also integer and float variables (discretized in a given range), see Variable.
