Quadratic Unconstrained Optimization#

Quadratic Unconstrained Binary Optimization (QUBO) is a mathematical optimization technique in which a business problem is formulated with help of binary variables as a quadratic cost function. A variable configuration which minimizes (sometimes also formulated as maximization problem) the quadratic cost function represents an optimal solution of the optimization problem. Since the corresponding quadratic polynomial represents the optimization problem, the latter is often called QUBO as well.

One core functionality of Digital Annealer Development Kit is the programming of QUBOs in form of BinPol objects.

Example Problem#

Consider the MaxCut problem with a graph \(G(V,E)\) where \(V=\{0,\ldots,N-1\}\) with \(N>0\) is the set of vertices and \(E\) is the set of edges. The goal is to find a maximal cut, i.e., to split the vertices into two sub-sets \(S \subset V\) and \(V \setminus S\) such that the number of edges in \(E\) for which the adjacent vertices are in different sets is maximized. To formulate this problem as a QUBO we first define for each \(i\in V\) a binary decision variable

\(\begin{equation} x_i = \left\lbrace\begin{matrix} 1, &\text{if vertex } i \in S \\ 0, &\text{if vertex } i \in V \setminus S \end{matrix} \right. \end{equation}\)

Variables and QUBO#

The quadratic polynomial which needs to be minimized (in order to maximize) the number of edges in the cut is formulated as

\(H = -\sum_{(i,j) \in E} \left(x_{i} - x_{j} \right) ^2\).

The variables \(x_i\) can be defined using a VarShapeSet object and the QUBO for the MaxCut problem using a BinPol object can be programmed as follows

from dadk.BinPol import *

N = 10
E = [(i, j) for i in range(N) for j in range(i + 1, N, 2)]
x_bit_array_shape = BitArrayShape(name='x', shape=(N,))
var_shape_set = VarShapeSet(x_bit_array_shape)
BinPol.freeze_var_shape_set(var_shape_set)
qubo = BinPol()
for i, j in E:
    qubo += -(Term(1, ('x', i)) - Term(1, ('x', j))) ** 2

Alternatively, the last three lines can be written as

qubo = BinPol()
for i, j in E:
    tmp_qubo = - BinPol().add_term(1, ('x', i)).add_term(-1, ('x', j)).power(2)
    qubo.add(tmp_qubo)

You have seen how to define a QUBO, add terms to it, how to add two QUBOs together, how to multiply a scalar to the QUBO and how to multiply two QUBOs with each other.

The scalar multiplication and the multiplication of two QUBOs might have been a bit hidden. Scalar multiplication was triggered by negation of the term \(\left(x_{i} - x_{j} \right) ^2\) for given \((i,j)\in E\). Multiplication of two QUBOs was done with the power method here. Multiplication of a scalar could also have been explicitly called by

tmp_qubo = BinPol().add_term(1, ('x', i)).add_term(-1, ('x', j)).power(2)
tmp_qubo.multiply_scalar(-1)

Instead of using the power method, one could also have multiplied the (linear) polynomial with itself

tmp_qubo = BinPol().add_term(1, ('x', i)).add_term(-1, ('x', j))
tmp_qubo.multiply(tmp_qubo)
tmp_qubo.multiply_scalar(-1)

The multiply method can be used to multiply a different or here the same (linear) BinPol object to a current (linear) BinPol object. Since the power method multiplies the BinPol to itself, it is implemented more efficient than the multiply method and should be preferred.
