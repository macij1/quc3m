Solving Constrained QUBOs#

This section contains code examples for solving different variations of the worker assignment optimization problem (using DADK’s QUBOSolverCPU), for interpreting the solutions, and some modeling advice to improve convergence of the solver.

Problem Formulation#

Recall the code for the two different variations of the worker assignment optimization problem as BinPol object qubo from the previous section.

With

minimize_workers = True

for the first example and

minimize_workers = False

for the second example, the model parameters as well as BitArrayShape objects for both variations are programmed as

from dadk.BinPol import *

days = ['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday']
D = len(days)

shifts = ['Early', 'Late', 'Night']
S = len(shifts)

workers = ['Adam1', 'Bob1', 'Charly1', 'Dave1', 'Earl1',
           'Adam2', 'Bob2', 'Charly2', 'Dave2', 'Earl2',
           'Adam3', 'Bob3', 'Charly3', 'Dave3', 'Earl3',
           'Adam4', 'Bob4', 'Charly4', 'Dave4', 'Earl4']
W = len(workers)

blocked = [('Monday', 'Night', 'Adam1'),
           ('Wednesday', 'Late', 'Charly2'),
           ('Friday', 'Early', 'Earl2')]

constant_bits_x_bits = np.full((D, S, W), -1, dtype=np.int8)
for day, shift, worker in blocked:
    constant_bits_x_bits[days.index(day), shifts.index(shift), workers.index(worker)] = 0

x_bit_array_shape = BitArrayShape(name='x', constant_bits=constant_bits_x_bits)

demand = {
    'Monday': {'Early': 2, 'Late': 5, 'Night': 3},
    'Tuesday': {'Early': 1, 'Late': 5, 'Night': 3},
    'Wednesday': {'Early': 2, 'Late': 5, 'Night': 3},
    'Thursday': {'Early': 1, 'Late': 5, 'Night': 3},
    'Friday': {'Early': 2, 'Late': 5, 'Night': 1}}

if minimize_workers:
    constant_bits_y_bits = np.full((W,), -1, dtype=np.int8)
else:
    constant_bits_y_bits = np.full((W,), 1, dtype=np.int8)
y_bit_array_shape = BitArrayShape(name='y', constant_bits=constant_bits_y_bits)

var_shape_set = VarShapeSet(x_bit_array_shape,
                            y_bit_array_shape)
BinPol.freeze_var_shape_set(var_shape_set)

Note

Note that we defined y_bit_array_shape for both optimization problems, but set all \(y\) variables to constant one in case of the objective to distribute shifts evenly among the workers. In that case, all workers have to be assigned to shifts. In the case of minimize_workers = False dropping the usage of y_bit_array_shape along with constant bits, would improve the performance.

The code for either of the optimization problems is programmed as

weight = 2000
penalty_qubo = BinPol()
for d, day in enumerate(days):
    for s, shift in enumerate(shifts):
        penalty_qubo.add_exactly_n_bits_on(n=demand[day][shift],
                                           bits=[('x', d, s, w) for w, worker in enumerate(workers)],
                                           scalar=weight)
for w, worker in enumerate(workers):
    for d, day in enumerate(days):
        penalty_qubo.add_most_1_bit_on(bits=[('x', d, s, w) for s, shift in enumerate(shifts)],
                                       scalar=weight)

if minimize_workers:
    for w, worker in enumerate(workers):
        for d, day in enumerate(days):
            for s, shift in enumerate(shifts):
                penalty_qubo.add_term(weight, ('x', d, s, w))
                penalty_qubo.add_term(-weight, ('x', d, s, w), ('y', w))

    weight = 1800
    qubo = BinPol()
    for w, worker in enumerate(workers):
        qubo.add_term(weight, ('y', w))

else:
    nr_shifts_per_worker = sum([demand[day][shift] for d, day in enumerate(days) for s, shift in enumerate(shifts)]) / W
    weight = 400
    qubo = BinPol()
    for w, worker in enumerate(workers):
        qubo_tmp = BinPol()
        qubo_tmp.add_term(nr_shifts_per_worker)
        for d, day in enumerate(days):
            for s, shift in enumerate(shifts):
                qubo_tmp.add_term(-1, ('x', d, s, w))
        qubo_tmp.power2()
        qubo.add(qubo_tmp, weight)

Note

We chose to not define the one_hot_groups for the VarShapeSet objects in this example. In general, defining one_hot_groups whenever possible probably improves convergence of the solver since within one hot groups, several bit flips are done in one iteration in order to maintain a state satisfying the one hot constraints before and after the iteration. For the examples in this section, there are three out of 15 shifts which only require one worker and can therefore be modelled as a one hot group of size W=20.

In the examples, there is no difference in the concrete worker assigned to a shift. Therefore, during the annealing process, changing the worker assigned to a shift which is programmed as a one hot group (which can be done in a single iteration) often leads to no change in the corresponding same energy value. For shifts which can not be programmed as a one hot group exchanging a worker can only be done in two iterations and after the first iteration, the energy value usually has to increase since a constraint has to be violated temporarily (after an initial annealing phase).

The annealing process is a randomized approach in which candidate bit flips with energy decrease or no energy change are much more likely than bit flips with energy increase. Therefore, using one hot groups in our examples slows down convergence of the solver because the annealing process gets caught in changing worker assignments within the one hot groups.

In general, programming with one hot groups is recommended especially if most or all bits are included in one hot groups.

Inequalities to limit the number of shifts a worker can be assigned to within a week are programmed as

upper_shift_limit = 4
inequalities = []
for w, worker in enumerate(workers):
    tmp_qubo = BinPol().add_term(-upper_shift_limit)
    for d, day in enumerate(days):
        for s, shift in enumerate(shifts):
            tmp_qubo.add_term(1, ('x', d, s, w))
    inequalities.append(Inequality(qubo=tmp_qubo, lambda_value=1000))

Choosing the Solver#

To solve either of the examples above, we use DADK’s QUBOSolverCPU:

from dadk.QUBOSolverCPU import *
solver = QUBOSolverCPU(number_runs=2,
                       number_iterations=10000,
                       temperature_sampling=True,  # default
                       random_seed=42,
                       scaling_action=ScalingAction.AUTO_SCALING)

Similar as in Basic Usage - Solving QUBOs, the code above defines a QUBOSolverCPU object which does two independent annealing runs (number_runs = 2) with 10000 iterations (number_iterations = 10000) each. The temperature settings for the annealing process are again determined automatically but could also have been set manually. Temperature decrease happens after every step according to the parameter temperature_interval = 1. Again, the parameter offset_increase_rate is determined automatically, i.e., when the random walk in the annealing process ends in a local minimum and no candidate can be determined for the next step, the energy is increased temporarily and stepwise by offset_increase_rate; this accelerates the escape from the local minimum. For offset_increase_rate = 0 the classical simulated annealing algorithm is applied. We have also set a random seed (random_seed = 42) in order to make the solutions reproducible. Since for minimize_workers = False the QUBO contains float values, we use the automatic scaling functionality of the dadk and set scaling_action=ScalingAction.AUTO_SCALING.

Either of the examples above is then solved by

solution_list = solver.minimize(qubo + penalty_qubo)

without the inequality constraints or by

solution_list = solver.minimize(qubo + penalty_qubo,
                                inequalities=inequalities)

with inequality constraints.

Analyzing and Interpreting the Solution#

Both solutions of the two independent runs are part of the SolutionList object which is returned by the method and stored in solution_list.

The Solution object with minimal energy is extracted by

solution = solution_list.min_solution

Shift assignments and information about the number of workers which have been assigned shifts can be retrieved and evaluated by

sol_y = solution['y'].data
print('Number of working workers: ', np.sum(sol_y))

sol_x = solution['x'].data
nr_shifts_per_worker = sum([demand[day][shift] for d, day in enumerate(days) for s, shift in enumerate(shifts)]) / np.sum(sol_y)
print('Shifts per working worker:', nr_shifts_per_worker)
shifts_of_workers = [np.sum(sol_x[:, :, w]) for w, worker in enumerate(workers)]
mean_deviation = np.mean([(shifts_of_workers[w] - nr_shifts_per_worker) ** 2 for w, worker in enumerate(workers) if shifts_of_workers[w] > 0])
print('Mean quadratic deviation of worker shift numbers from mean number of shifts: ', mean_deviation)

Without inequalities and minimize_workers = True, this results in 11 workers which have been at least one assigned shift and the mean quadratic deviation of worker shift numbers from the mean number of shifts per working worker is 0.51. With additional inequalities, the number of workers which have been assigned at least one shift increases to 12 and the mean quadratic deviation of worker shift numbers from the mean number of shifts per working worker is 0.14.

Without inequalities and minimize_workers = False, all workers (20) are assigned shifts as a result of the objective. The mean quadratic deviation of worker shift numbers from the mean number of shifts per working worker is 0.21. With additional inequalities, the solution remains the same, since already without inequalities the maximal number of shifts assigned to a worker was three.

Note

A mean quadratic deviation of worker shift numbers of 0.14 from the mean number of shifts per working worker has been the best among all tests, even without any direct objective to distribute shifts evenly among the workers. This comes from the fact that, in order to minimize the number of working workers, all of the working workers except two are assigned the maximal possible number of shifts upper_shift_limit = 4. The other two working workers are assigned three shifts which leads to an optimal distribution of shifts among workers. The shifts among the latter two workers could also have been distributed differently with a different random seed since even distribution of work among workers was not part of the objective.

The solution for the test with minimize_workers = False was also optimal for the respective optimization problem but was forced to use all the workers by demanding a number of assigned shifts of nr_shifts_per_worker = np.sum(A)/W for each worker in the objective.

To summarize, in modeling an optimization problem mathematically one has to be very careful in choosing objectives and constraints such that the optimal solution of the problem is actually optimal for the original business problem.
